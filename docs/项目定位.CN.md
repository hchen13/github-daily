# GitHub Daily 项目定位

中文名：辛哥的开源风向
英文名：GitHub Daily
目录名：`~/projects/github-daily/`

## 定位

这是一个刊物项目，不是某个 agent 的插件壳。

它的核心职责只有一件事：稳定地产出可发布的日报。

## 为什么要 agent-independent

因为分发层会变，运行时也会变。

今天可能是 OpenClaw 发，明天可能是 Hermes 发，后天也可能直接本地 cron 生成 Markdown 再推到飞书。刊物本身应该独立存在，任何 agent 都只是下游消费者。

## 结构原则

1. 采集层独立
   - repo 深度跟踪
   - trending 热点发现

2. 分析层独立
   - 可以先用 Claude Code CLI
   - 以后也可以切到 API、Hermes、别的模型
   - 上层接口保持稳定

3. 出刊层独立
   - 标准产物优先是 Markdown + JSON
   - 发布消息只是出刊结果的不同包装

4. 分发层独立
   - 飞书、Webhook、邮件、命令行输出都放在下游

## 内容结构

日报固定两大板块：

### 1. AI Agent 标杆项目动态
跟踪四个项目：
- Claude Code
- Codex
- OpenClaw
- Hermes Agent

回答三个问题：
- 用户在遇到什么问题
- 社区在补什么能力
- 官方在押什么方向

### 2. GitHub 开源社区风向
跟踪：
- Daily Trending Top 3
- Weekly Trending Top 10

回答两个问题：
- 现在开发者社区最热的主题是什么
- 这些热点里哪些是持续趋势，哪些只是短期噪音

## 当前结论

这个项目应该放在 `~/projects/github-daily/`。

Pulse 可以复用其中一部分能力，但 GitHub Daily 本身应该作为独立刊物工程存在。
