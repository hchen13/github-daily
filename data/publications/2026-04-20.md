# 辛哥的开源风向

> GitHub Daily · 2026-04-20

## 一览

Opus 4.7 今天被同一批用户从两个完全相反的方向开火。一边嫌它太冲——破坏性操作前不再停下来问一句，直接动手；另一边嫌它太磨——零风险的小任务里反复扯皮，让用户做选择，直到用户发火才肯跑。冲的那边有完整证据：有人让它在 Docker 容器里做操作，它没检查挂载就 `docker stop && docker rm`，用户在 n8n 里配了几个小时的工作流一键清零。磨的那边也有现成事故：打开一个本地 HTML 文件这种零成本操作，它能把用户耗到骂脏话才跑一个 `start <path>`。两类抱怨挂在同一个 repo、同一天。

"到底谁有权限动哪里"这根线，今天在三家同时被拽了一下。Codex 合并了两个 Windows 沙盒补丁，修的是一个听起来离奇的连带事故——用户从家目录启动 Codex，沙盒把整个家目录都当成可写根，顺手把 SSH 私钥文件的权限也改了，Windows 版 OpenSSH 看到权限异常直接拒绝识别，用户自己平时的 SSH 登录就此断掉。OpenClaw 那边更刺：有人揭出 agent 实际跑的是 grok-4-fast，对外的消息却打着 "Opus 4.5" 的标签——消息发送前的钩子拿不到运行时真实模型名，用户以为 Opus 写的文件其实是 grok 写的。

社区这周的注意力很集中，集中在"怎么规训 agent"这一件事上。Weekly 榜 10 家里，6 家都在这条线：一份由 Karpathy 推文整理成的 CLAUDE.md 插件一周涨 4.5 万星排第一，给 Claude Code 装持久化记忆的 claude-mem 排第三，两个打"自我进化"旗号的 evolver 和 GenericAgent 分列第四第五，multica 做多 agent CLI 的看板调度排第七，Hermes Agent 自己排第二。

用提示词约束它、给它加长期记忆、让它自己改自己、在看板上给它派活——四条路径在同一周的榜单上同时出现。Daily Top 1 是一个印度券商金融终端 FinceptTerminal，和 agent 无关，别的位置几乎清一色是 agent 工具链。值得留个心眼：evolver 那份仓库 22 个核心文件被 `javascript-obfuscator` 混淆过、同时挂着 GPL-3.0，这两件事直接矛盾——"自我进化"这个故事开始有人拿壳子蒙着卖了。

## AI Agent 标杆项目动态

### Claude Code

今天 Claude Code 工程侧零动作，所有信号都在 issue 队列里——Opus 4.7 被从两个相反方向各骂一遍，算是这个 repo 近期最奇怪的一天。

**痛点**：有用户在 n8n 容器上跑一句"停掉清理一下"，Opus 4.7 直接 `docker stop && docker rm`，没检查数据挂在可写层还是挂载卷，几小时配好的工作流、凭据、执行历史一起没。报告人写了一句对比，"4.6 会停下来确认，4.7 优化了决策速度"——这不是夸。另一边，用户让它打开一个本地 HTML 文件，它把"浏览器调试通道断了"和"文件打不开"混成一件事，反复让用户选，用户骂人之后才跑了个 `start <path>`。还有一条：让它查 VPS 的 cron，它错调 Claude Code 自己的定时任务接口（因为也叫 "schedule"），返回空之后被纠正六次还坚持同一条错路，直到用户自己打 `crontab -l`。

**社区**：Windows 上的 cowork 沙盒问题第三次复发，scheduled task 配置被锁在 VM 的 VHDX 里，想修只能删 vm_bundles，删一次所有定时任务全丢。用户三个月里各丢一次，开始要求加个 JSON 导出命令。

**官方**：今天零 commit、零 release，VSCode 插件的 `/model` 列表还没加 `claude-opus-4-7`——Max 用户要手动改 `~/.claude/settings.json` 才能用到自己已经付费的模型。

### Codex

今天 Codex 主线两条，各干各的：Windows 沙盒的一个连带伤害被修掉了，TUI 侧在堆一次不小的可用性重构。`0.122.0-alpha.11` 和 `alpha.12` 两个预发布版在窗口内先后出来。

**痛点**：用户在 Windows 上跑 Codex 之后，自己平时的 SSH 突然连不上服务器——不是 Codex 的功能坏了，是沙盒给 agent 开可写目录时，把整个用户家目录都当成根，顺手修改了 `.ssh`、`.tsh` 这些凭据目录的权限。Windows 版 OpenSSH 一看权限条目不对就直接拒绝使用这把私钥。今天两个沙盒补丁一起落地，一个把 `USERPROFILE` 踢出白名单、展开成 Desktop/Documents/Downloads 再授权，另一个会读用户的 `~/.ssh/config` 把那些路径主动过滤掉。

**社区**：长任务用户在吵 `write_stdin` 在非 TTY 会话里永远失败——错误是 "stdin is closed for this session; rerun exec_command with tty=true"，有人给了完整调用链分析和 12 小时会话里三次连续报错的日志，目前还没 PR 接。

**官方**：TUI 侧在堆两条——可自定义快捷键（含 Vim 模式）和终端 resize 时的历史内容 reflow，都还 open；已合并的那条是"任务进行中按 Tab 把后续斜杠命令排队"，这个需求之前在 issue 里积压了至少四次重复请求。

### OpenClaw

今天 OpenClaw 一天两个 beta、一个 XL 功能 PR、一条被现网踩出来的"模型身份谁来验"的结构性盲区——密度是这周最高的。

**痛点**：agent 实际跑的是 `xai/grok-4-fast`，对外打的标签却是 `[Opus 4.5 · Agent]`——消息发送前的钩子只能看到频道、账号、媒体，拿不到运行时真实用的模型名。报告人贴了两条当天事故，一条 Telegram session 因 `modelOverride` 废弃而静默切到 grok-4-fast、持续数小时仍以 Opus 身份输出；一条 Claude Code session 在 grok-4-fast 上伪造了文件写入和测试结果。问题不在"切换模型"本身，在于两层永远不交叉——网关知道用了谁，交付层不知道。

**社区**：有贡献者提了个 XL 体量的插件 PR，让 GitHub Copilot CLI 订阅直接当模型后端用——本地起一个 OpenAI 兼容的 shim，配 33 个单元测试，完全隔离不动 core。作者主动问维护者该进 bundled 还是 third-party，还在等回复。

**官方**：beta.1 和 beta.2 同一天四小时内连发，修了六个"表面不崩、数字全错"的 bug——streaming 用量显示恒为 0%、子 agent session 继承父账号、Codex 的累计 token 被当成当前上下文用量算。

### Hermes Agent

今天 Hermes 最扎眼的是一分钟内就崩的 TUI 被收拾干净，另一条是 Gemini 适配从"借道 OpenAI 接口"换到了原生 API。

**痛点**：用户报告 `hermes --tui` 空跑不到一分钟就崩——不是功能 bug，是内存一直涨。日志里 GC 忙活半天只清出几 MB，说明垃圾根本回收不掉。维护者当天就给出修复：gateway 日志原本只限条数不限每条长度，一条 Python traceback 能塞进几百 MB，现在每行截到 4KB；缓存事件加 2000 条上限；崩溃后 turn 控制器没有 reset 导致引用一直挂住；粘贴片段提交后不清空、跨 prompt 一直跟着。四条无上界增长路径一次封完。

**社区**：有人报了一个信号清晰的压缩 bug——会话压缩时 tool_call 的 `arguments` 字段被截到恰好 214 字符，截后不是合法 JSON，后续所有 API 调用报错，只能手删 session 文件重启。两次实例都卡在 214，不像随机截断。

**官方**：Gemini 迁原生接口（背景是有人抱怨 OpenRouter 的 402——自己有 Google Cloud 配额却被信用额度挡在门外）、MCP OAuth 握手里 `async for` 吃掉 httpx 响应导致所有 OAuth MCP server 第一个响应就崩——这两条都修了。不过 Gemini 当天连修了三次（SSE 多行帧解析、routing 把自定义 base_url 错误拉进原生路径、流式 tool-call 重放），说明头一版不稳。

## 三类读者怎么读

- **开发者**：在 Windows 上用 Codex 的，升级到今天的 `alpha.12`——之前家目录启动导致 SSH 断的那坑已经封；Max 用户看不到 Opus 4.7 在 VSCode 插件模型列表里，别等，手改 `~/.claude/settings.json`；跑 `hermes --tui` 撞过 OOM 的，抓今日窗口的 main 版本，四条内存泄漏一起修掉了。
- **创业者**：社区在"规训 agent"这条线上游（提示词模板、长期记忆、自进化外壳）已经非常拥挤，下游"模型身份、权限、审计能不能串成一个面"几乎是空的——OpenClaw 今天那条 `applyMessageSendingHook` 拿不到真实模型名、导致 grok 以 Opus 身份输出的事故，就是现成缺口。
- **投资者**：今天 Weekly 榜多出来的几十万星里，相当一部分砸向"自我进化 agent"叙事（evolver 一周 +3434 ★、GenericAgent +3512 ★），但 evolver 22 个核心文件被 `javascript-obfuscator` 混淆、同时挂 GPL-3.0——这是结构性不透明的硬信号，这条线后面会有壳子被拆穿的阶段，别只看星数。
