## AI Agent 标杆项目动态

### Claude Code

当前处于激进迭代期，风险与修复并行。v2.1.113 将执行层切换为原生二进制，Nix/Guix 等非标准 Linux 发行版因 ELF 解释器路径不匹配即刻出现回归，v2.1.114 当天跟进热修复，顺带修复了 Agent Teams 权限对话框崩溃。安全侧同日浮现两条告警：claude-in-chrome MCP 可跨账号无提示控制他人浏览器；文件历史快照将 `.env` 私钥明文留存于本地隐藏目录。多 agent 并发下的父会话死锁问题也有详细复现进入排队。

### Codex

今天 Codex 最集中的动作是设备密钥基础设施的批量落地——四个 PR 一次性覆盖协议定义、核心 crate、app-server 适配层，以及 macOS（Secure Enclave）、Windows（CNG/TPM）、Linux（TPM2）三套平台实现，目标是为本地设备绑定认证提供硬件级支撑。MCP 工具的 permission hook 同步推进，将原有 Bash hook 的 payload 结构延伸至 MCP 工具调用。Plan Mode 新增"清空上下文后实现"路径，让规划对话与实现线程彻底解耦。24 小时内连发三个 alpha 预发布（alpha.8/9/10），团队明显处于密集冲刺阶段。

### OpenClaw

今天 OpenClaw 以稳定性修复为主线。weixin 插件 sidecar 误杀父级 gateway 进程、触发无限重启循环的高危 bug 当天提、当天合入，修复方式是在 stale-pid 清理逻辑中排除整条祖先 PID 链。cron 任务超时后遗留幽灵 task 行的生产级问题（29 条 stale_running 任务）也排队等合。功能侧有一个大 PR 进入评审：引入实验性 prompt 覆盖层，支持 model-specific AGENTS.md addenda，首批含 GPT-5.4 调参包。当前影响面最宽的未修问题是 OpenRouter base URL 写成 `/v1` 而非 `/api/v1`，导致所有模型调用静默失败。

### Hermes Agent

今天 Hermes Agent 的两处主动架构决策值得记录。其一，新增 `/steer` 指令，允许用户在 agent 运行期间于下一次工具调用结束后注入引导文本，不中断流程也不破坏 prompt cache。其二，`execute_code` 默认执行模式从隔离临时目录切换为 project 模式，与 `terminal()` 共用相同 CWD 和解释器，直接解决弱模型反复混淆工作目录的问题。稳健性修复同步密集落地：流式传输中途 stall 时工具调用不再静默丢失，Signal typing 指标无限重试被指数退避压制，Telegram session 竞态 bug 当天报告当天提 PR。另有一支分支明确标注"灵感来自 Claude Code 2.1.113"，将 macOS `/private/` 路径绕过等三条危险命令检测规则移植进来——跨项目借鉴信号清晰。

### Daily Trending Top 1 · thunderbird/thunderbolt

Thunderbird 官方出的跨平台 AI 客户端，基本盘是严肃工程而不是 demo：React 19 + Tauri 2 六端打包、PowerSync 离线同步、better-auth + 后量子密钥、Drizzle + PGlite、Docker/K8s/Pulumi 全套部署，MPL-2.0，测试文件满地跑。这套配置明显奔着企业私有化部署去的。但"AI 自主可控"目前更像愿景——README 自己承认强依赖后端认证和搜索服务，离线 first 还没做到，跑全套得自己拉一个 PowerSync + Postgres 后端，门槛不低。Mozilla 品牌是最大加分项；看点在于它愿不愿意把这个做到 Thunderbird 那个量级。

## 三类读者怎么读

- **开发者**：Claude Code 正在激进迭代，如果你跑在 Nix/Guix 或非标准 Linux 上，锁定版本比跟最新更安全；同时检查你的 `.env` 是否已被历史快照留存在本地隐藏目录。
- **创业者**：thunderbolt 和 multica 都在用"企业私有化 AI"打差异化，这个方向有市场，但现在没有一家真正把部署门槛做到 one-click——谁先做到，谁先拿到下一轮企业采购。
- **投资者**：Hermes 本周 +47K 星、Codex 密集冲刺硬件级认证、Claude Code 修安全漏洞——agent 基础设施的竞争已从功能扩张转向安全与可信边界，下一个值得看的赌注是"可审计、可控制的 agent 运行时"，而不是又一个 wrapper。
