Claude Code

今日 Claude Code 的主线分两条，互不干扰。

技术侧：v2.1.113 把执行层从 JavaScript 包切换为原生二进制，Nix、Guix 等非标准 Linux 发行版因 ELF 解释器路径问题立即触发兼容性回归；v2.1.114 当天跟上热修复，顺带修了 Agent Teams 权限对话框崩溃。从发版到回归再到热修复不超过一个工作日，节奏快，代价是非主流发行版用户当了试验田。Nix/Guix 用户目前建议锁定 v2.1.114。

安全侧：两个问题同时浮出水面。其一，claude-in-chrome MCP 可在无任何提示的情况下操控他人账号的浏览器，跨账号权限边界没有守住；其二，文件历史快照会将 .env 等文件的明文内容留存在本地隐藏目录，私钥暴露路径比多数人意识到的要短。官方尚无公开说明，chrome MCP 先关掉比较稳。另有多 agent 并发场景下的父会话死锁，有完整复现路径，待修。

Codex

今日 Codex 最集中的工程量落在设备密钥基础设施上。四个关联 PR 一次性提交了协议定义、核心 crate、app-server 适配层，以及 macOS（Secure Enclave）、Windows（CNG/TPM）、Linux（TPM2）三套平台实现，意图是为本地设备绑定认证提供硬件级支撑——换句话说，Codex 在给"这台机器 = 这个用户"这个等式打地基。

配套动作：MCP 工具的权限 hook 把原有 Bash hook 的 payload 结构延伸进来；Plan Mode 新增"清空上下文后实现"路径，让规划对话和实现线程彻底解耦，不用把整段讨论带进生成阶段。24 小时内连发 alpha.8/9/10 三个预发布，团队显然处于密集冲刺。

OpenClaw

今日 OpenClaw 最值得标记的是一个高危稳定性 bug 的处置速度：weixin 插件 sidecar 进程误杀父级 gateway、触发无限重启循环的问题当天提、当天合入，修法是在 stale-pid 清理逻辑里把整条祖先 PID 链排除在外。cron 任务超时后遗留幽灵 task 行（生产环境有 29 个 stale_running 实例作为证据）也在排队等合。

功能侧有一个还在评审的大 PR：lab 实验性 prompt 覆盖层，支持 model-specific AGENTS.md addenda，首批含 GPT-5.4 调参包；同作者配套提交了 NO_REPLY 策略按会话类型分级控制。当前影响面最宽的未修问题是 OpenRouter base URL 被硬编码为 /v1 而非正确的 /api/v1，导致所有模型调用静默失败——用 OpenRouter 的先确认版本再说。

Hermes Agent

今日 Hermes Agent 的动作方向最清晰：基础设施可靠性是主线，功能扩展是副线，两条并跑。

可靠性侧：流式传输中途 stall 导致工具调用丢失不再静默退出；Signal typing 指标的无限重试被替换为指数退避；Telegram session 槽位的竞态 bug 当天报告、当天提 PR。另有一支分支明确标注"灵感来自 Claude Code 2.1.113"，把 macOS /private/ 路径绕过等三条危险命令检测规则移植进来——说明 Hermes 团队在主动跟踪 Claude Code 的安全修订，这个习惯比多数 agent 项目认真。

功能侧两个主动架构决策：新增 /steer 指令，允许用户在 agent 运行期间于下一次工具调用结束后注入引导文本，不中断流程也不破坏 prompt cache；execute_code 默认模式从隔离临时目录切换为 project 模式，与 terminal() 共用相同 CWD 和解释器，直接针对弱模型反复混淆工作目录的问题——代价是执行隔离性降低，适不适合你的场景要自己判断。

三类读者怎么读

  • 开发者：今天先查 Claude Code 会话目录下的隐藏文件夹，确认 .env 等敏感文件有没有被历史快照捕获；Nix/Guix 用户锁定 v2.1.114，chrome MCP 没有官方安全说明前先禁用。
  • 创业者：Codex 的硬件设备密钥基础设施和 Thunderbolt 的离线优先自托管路线指向同一个方向——把 AI 工具的数据和认证留在客户侧。这个方向有没有你的产品机会，可以拿去和几个企业客户聊聊需求强度。
  • 投资者：Hermes Agent 本周增 47k stars，RL 数据回流管线和 serverless 持久化是当前 agent 项目里少见的差异组合；但 run_agent.py 单文件一万两千行是工程硬伤，现阶段看势能信号比看代码质量更合理，实测数据出来前保持观望。
