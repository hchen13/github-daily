Claude Code

今日 Claude Code 最集中的事件是一次迁移代价：v2.1.113 将执行层从 JavaScript 包切换为原生 ELF 二进制，Nix、Guix 等非标准 Linux 发行版因解释器路径不匹配即刻出现兼容性回归，当日 v2.1.114 热修复跟进，同步修了 Agent Teams 权限对话框崩溃。迁移快、回滚快，但 non-standard Linux 的兼容性显然没有提前覆盖到。安全侧两条告警见今日头条；另有多 agent 并发下的父会话死锁已有详细复现在追踪，结论暂时看不准。

Codex

今日 Codex 最集中的信号是设备密钥基础设施的集中落地——四个关联 PR 一次性把协议定义、核心 crate、app-server 适配层以及 macOS（Secure Enclave）、Windows（CNG/TPM）、Linux（TPM2）三套平台实现全部提交，目标是把本地设备绑定认证推到硬件级。同日 MCP 工具的 permission hook 集成在积极推进，Plan Mode 新增了"清空上下文后实现"分支，让规划对话与实现线程彻底解耦。24 小时三个 alpha 预发布，节奏说明团队处于密集冲刺阶段，但距离可用状态还有多远看不准。

OpenClaw

今日 OpenClaw 最集中的信号是两处生产级 bug 的定损与修复。weixin 插件 sidecar 误杀父级 gateway 进程并触发无限重启循环，当天提、当天合入，修法是在 stale-pid 清理逻辑中排除整条祖先 PID 链，根因定位清晰。cron 超时遗留幽灵 task 行（已积累 29 条 stale_running 记录）的修复排队等合。当前影响面最宽的未修问题是 OpenRouter base URL 硬编为 /v1 而非 /api/v1，导致所有 OpenRouter 模型调用静默失败——用 OpenRouter 路由的先手动确认，当前版本不报错。功能侧，支持 model-specific AGENTS.md addenda 的实验性 prompt 覆盖层大 PR 进入评审阶段。

Hermes Agent

今日 Hermes Agent 最清晰的动向是两处主动的架构调整：新增 /steer 指令，允许在 agent 运行期于下一次工具调用结束后注入引导文本，不中断流程也不破坏 prompt cache；execute_code 默认执行模式从隔离临时目录改为 project 模式，与 terminal() 共享相同工作目录和解释器，解决弱模型反复混淆 CWD 的问题。可靠性侧密集落地：流式传输中途 stall 导致工具调用丢失不再静默退出，Signal typing 指标无限重试被指数退避压制，Telegram session 竞态当天报告当天提 PR。另有一条明确标注"灵感来自 Claude Code 2.1.113"的分支，将 macOS /private/ 路径绕过等三条危险命令检测规则移植进来——跨项目借鉴，方向清晰。

GitHub 开源社区风向

Daily Trending Top 1 · thunderbird/thunderbolt

thunderbolt 是 Thunderbird 官方团队出的跨平台 AI 客户端，不是社区玩具。技术栈一句话：React 19 + Tauri 2 打六端壳、Bun + Elysia 后端、PowerSync 做离线同步、better-auth + 后量子加密、Drizzle + PGlite、Playwright e2e、Docker/K8s/Pulumi 部署全家桶——版本号已到 0.1.87、PR 编号超过 600，迭代时间和密度都不短。

定位上奔着企业私有化部署：自带模型 API key、接 Ollama/OpenAI 兼容端点、MPL-2.0 开源。但现阶段"自主可控"更多是愿景——README 自承强依赖后端认证和搜索服务，真跑起来得自己拉一套 PowerSync + Postgres，门槛不低。和 LibreChat、Jan 这类已经铺开的项目比，技术差异点主要靠 Thunderbird 品牌和企业 FDE 支持，没有特别亮眼的新意。官方声明安全审计仍在进行中，这点使用前要知道。
